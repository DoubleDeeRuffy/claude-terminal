## Key Accomplishments

1. **Tab context-menu infrastructure** — Right-click tab menu with Rename, Close, Close Others, Close Tabs to Right (Phase 27), plus on-demand **AI Rename** via Haiku with async loading indicator (Phase 34)
2. **Full tab + Claude session persistence** — File and terminal tabs restored across restarts with exact ordering + active state (Phase 26); doubled-linebreak paste and Enter-key normalization fixed (Phase 28)
3. **Heartbeat split: user time-tracking vs. Claude activity** — Separated per-project user heartbeat from per-terminal Claude idle detection; 10s tick; configurable 15s/30s/1min/…/10min idle thresholds (Phase 29 + 29.1)
4. **NSIS silent install + configurable auto-updater** — `/S` flag respected, `.silent-install` marker applies defaults without wizard (Phase 30); updater settings UI with check interval, download mode, install mode + manual download banner (Phase 33)
5. **Terminal rendering stability** — Debounced scroll preservation (80ms settle) + rapid-output guard suppressing `terminal.clear()` during Claude TUI redraws — fixes flicker, buffer loss, and blackouts on tab switch (Phase 36)
6. **Rider-quality git tab** — `getBranchesWithTracking` + `getRecentBranches` git utilities, hierarchical branch treeview with Recent/Local/Remote sections + ahead/behind badges + search, resizable commit graph modal with 8-handle resize and persistent dimensions + filter toolbar, top-bar branch/pull/push status sync + periodic 5-minute refresh (Phase 37)
7. **Clipboard image paste into terminal** — Ctrl+V intercepts images, inline thumbnail preview bar, temp file save with 1-hour TTL cleanup, path injection into Claude CLI prompt on Enter (Phase 38)
8. **UX polish** — Fix display percentages in usage panel (Phase 35), flex-based sizing on empty-pane controls fixing overlap/disabled state (Phase 39), pane divider fixes and constraint tweaks (Phase 25), close warning when Claude is actively working (Phase 32)

## Known Gaps

Milestone shipped with the following intentional gaps, all addressed by the successor milestone v1.2:

- **Phase 29 (Adjust-Idle-Recognization)** — UAT passed but PR still pending upstream. Functionality shipped on the fork.
- **Phase 29.1 (Bugfix-Idle-Recognition-Cross-Tab)** — Not yet completed. Non-blocking; tracked as tech debt.
- **Phase 31C / 31D (Tab-Splitview)** — Split triggers (31C) and layout persistence (31D) **intentionally abandoned**. The entire split-pane / PaneManager subsystem is being removed by **v1.2 Phase 2 (Remove Split-Pane / PaneManager)**, so extending it further was wasted effort. Phases 31A (infrastructure) and 31B (pane-aware activation) shipped.
- **Phase 32 (Close-Warnings)** — SUMMARY.md exists (implementation landed) but plan checkbox was never flipped. Effectively shipped; status-tracking gap only.
- **Phase 25 / 26 / 28** — Shipped cleanly (all checkboxed in roadmap) but no SUMMARY.md was written. Commit history is the authoritative record for these phases.

## Forward Path

**v1.2 — Tab System Rewrite** (scaffolded in parallel, executes next) will:
- Delete `PaneManager.js` + all split-pane code (replaces 31C/31D)
- Rebuild the tab subsystem with string IDs, v3 persistence format, and v2→v3 migration
- Fix the three recurring tab bugs (rename wrong tab, wrong tabs on restore, wrong names on restore) at their root cause

See `.gsd/milestones/1.2/ROADMAP.md` for the full v1.2 roadmap.

## Milestone Summary

**Delivered:** Broad consolidation pass across tab UX, persistence, updater, terminal rendering, git tab, and clipboard — plus the infrastructure groundwork that made the v1.2 tab-system rewrite practical.

**Key Decisions:**
- **Abandon split-pane completion (31C/31D)** in favor of removing the entire PaneManager subsystem in v1.2 — the fork owner doesn't use split-pane, and extending it further would be dead code.
- **Keep the pane-aware primitives (31A/31B)** shipped — they were cheap to land and exercised the DOM container abstraction that informed the rewrite.
- **Ship Phase 37 with a WIP commit** (`52d9d17a`) rather than blocking the milestone on the remaining polish — v1.2 work started in parallel in a separate worktree.
- **Fork-only milestone** — 6 upstream PRs remain open for earlier phases; v1.1 work landed on the fork's `main` without waiting for upstream review.
