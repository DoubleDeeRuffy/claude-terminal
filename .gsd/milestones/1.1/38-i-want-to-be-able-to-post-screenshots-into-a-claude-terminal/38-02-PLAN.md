    **Gap 1 — Duplicate paste guard:**

    Add a module-level timestamp variable near line 84 (next to the existing `lastPasteTime`):

    ```javascript
    let lastImagePasteTime = 0;
    ```

    In `pasteWithImageCheck()` (line ~849), after `addTerminalImage(terminalId, file)` succeeds (line ~863), set the guard:

    ```javascript
    lastImagePasteTime = Date.now();
    ```

    In `setupPasteHandler()` (line ~910), inside the paste event handler, BEFORE processing image items (line ~918), add an early return if pasteWithImageCheck already handled it:

    ```javascript
    if (Date.now() - lastImagePasteTime < 1000) {
      return; // Already handled by pasteWithImageCheck in keydown
    }
    ```

    This goes right after the `if (inputChannel === 'terminal-input' && e.clipboardData) {` check, before the `const imageItems = ...` line.

    **Gap 2 — Path injection fix:**

    In the Enter intercept block (line ~2407-2418), make two changes:

    1. Convert Windows backslashes to forward slashes in file paths. Change the pathsStr construction from:
       ```javascript
       const pathsStr = filePaths.map(p => p.includes(' ') ? `"${p}"` : p).join(' ');
       ```
       to:
       ```javascript
       const pathsStr = filePaths.map(p => {
         const fwd = p.replace(/\\/g, '/');
         return fwd.includes(' ') ? `"${fwd}"` : fwd;
       }).join(' ');
       ```

    2. Change the input sending sequence. Instead of sending paths appended after user text in one shot (`api.terminal.input({ id, data: ` ${pathsStr}\r` })`), send JUST the paths followed by Enter. The user's text is already in the PTY input buffer from their typing. The current code sends a space + paths + \r which appends AFTER whatever xterm already sent for the Enter key. But the `return` at the end of the block prevents `api.terminal.input({ id, data })` from firing (line 2438), so the original Enter `\r` from `terminal.onData` is NOT sent to PTY.

    The correct sequence is: the user has typed text, images are pending, user presses Enter. We need to send: space + paths + \r. This IS what the code does. The issue is likely that the paths use backslashes which Claude CLI may not parse as file references.

    Add a `console.log` for debugging:
    ```javascript
    console.log('[TerminalManager] Injecting image paths:', pathsStr);
    ```

    Place it right before the `api.terminal.input` call.
