## Component Inventory

### 1. Terminal Image Preview Bar (`.terminal-image-preview`)

**Position:** Directly above the xterm.js terminal viewport, inside the terminal tab content area. Horizontally spans the full terminal width. Slides down from the top edge with a 150ms ease transition.

**Layout:** Flex row, `gap: 8px`, `padding: 8px 12px 4px`, `flex-wrap: wrap`. Matches `.chat-image-preview` layout exactly.

**Visibility:** `display: none` by default. Set to `display: flex` when one or more images are pending.

**Background:** `var(--bg-tertiary)` with `border-bottom: 1px solid var(--border-color)`.

**Max height:** 80px (single row of 64px thumbnails + 8px top padding + 4px bottom padding + 1px border). No scrolling — 5-image max enforced at paste time.

### 2. Terminal Image Thumbnail (`.terminal-image-thumb`)

**Dimensions:** 64x64px (matches `.chat-image-thumb`).

**Border:** `1px solid var(--border-color)`, `border-radius: 8px` (var(--radius)), `overflow: hidden`.

**Image fit:** `object-fit: cover` on the `<img>` child.

**Hover:** Shows remove button (opacity 0 to 1, 150ms ease).

### 3. Terminal Image Remove Button (`.terminal-image-remove`)

**Position:** Absolute, `top: 2px`, `right: 2px` inside the thumbnail.

**Dimensions:** 18x18px circle (`border-radius: 50%`).

**Background:** `rgba(0, 0, 0, 0.7)`.

**Icon:** Inline SVG "X" or text character "x", `color: #fff`, `font-size: 12px`.

**Interaction:** `opacity: 0` by default, `opacity: 1` on parent `.terminal-image-thumb:hover`. Transition: `opacity 0.15s ease`.

**Cursor:** `pointer`.

### 4. Image Count Badge (`.terminal-image-count`)

**Position:** Inline after the last thumbnail in the preview bar.

**Dimensions:** Auto-width pill, `padding: 2px 6px`, `border-radius: 10px`.

**Background:** `var(--accent-dim)` (rgba(217,119,6,0.15)).

**Text:** `var(--font-2xs)` (10px), `font-weight: 600`, `color: var(--accent)`. Format: "{count}/5".

**Visibility:** Only shown when 2 or more images are attached.

## Interaction Contract

### Paste Interception

| Trigger | Behavior |
|---------|----------|
| Ctrl+V with image in clipboard | Intercept before xterm.js processes the paste. Extract image data from `ClipboardEvent.clipboardData.items`. Add to pending images array (max 5). Show preview bar. |
| Ctrl+V with text in clipboard | Pass through to xterm.js as normal — no interception. |
| Ctrl+V when 5 images already pending | Ignore the image paste silently. The count badge already shows "5/5". |

### Image Addition Flow

1. User presses Ctrl+V with image on clipboard.
2. Renderer reads `ClipboardEvent.clipboardData.items`, finds `type.startsWith('image/')`.
3. Convert to blob, create `URL.createObjectURL()` for preview thumbnail.
4. Append thumbnail to `.terminal-image-preview`, show the bar.
5. Store blob reference in a pending images array on the TerminalManager instance (per-terminal, not global).

### Image Removal

| Trigger | Behavior |
|---------|----------|
| Click remove button on thumbnail | Remove that image from pending array. Revoke object URL. Remove thumbnail DOM element. If no images remain, hide preview bar. |

### Image Sending

| Trigger | Behavior |
|---------|----------|
| User presses Enter to send a prompt in the terminal | Before sending the text, save all pending images as temp files via IPC (`terminal:save-clipboard-image`). Inject file paths into the prompt text. Clear the preview bar. Revoke all object URLs. |

### Temp File Convention

- Path: `~/.claude-terminal/temp/screenshot-{timestamp}-{index}.png`
- IPC handler saves the image buffer, returns the absolute file path.
- Injected into prompt as space-separated file paths appended after the user's text.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Preview bar label | (none — thumbnails are self-explanatory, matching chat behavior) |
| Image count badge | "{count}/5" (e.g., "3/5") |
| Empty state heading | not applicable — preview bar is hidden when empty |
| Empty state body | not applicable |
| Error state | "Failed to read clipboard image" — shown as a toast via existing Toast component |
| Max images reached | No explicit message — badge shows "5/5", further pastes are silently ignored |

No destructive actions in this phase. Removing a thumbnail is lightweight and instant (no confirmation needed).

---

## State Contract

| State | Visual |
|-------|--------|
| No images pending | Preview bar hidden (`display: none`). Terminal occupies full height. |
| 1 image pending | Preview bar visible (80px). Single thumbnail. No count badge. |
| 2-5 images pending | Preview bar visible. Thumbnails in row. Count badge shown ("2/5" through "5/5"). |
| Sending (images being saved) | Thumbnails briefly show a subtle opacity reduction (0.5) during save. Preview bar clears after save completes. |
| Error (clipboard read failed) | Toast notification: "Failed to read clipboard image". Preview bar unchanged. |

## Accessibility

| Concern | Implementation |
|---------|---------------|
| Remove button keyboard access | `tabindex="0"`, `role="button"`, `aria-label="Remove image {n}"` |
| Preview bar announcement | `aria-live="polite"` on `.terminal-image-preview` so screen readers announce image additions |
| Image alt text | `alt="Screenshot {n}"` on each thumbnail `<img>` |

## CSS File Target

All new styles go in `styles/terminal.css` under a new section header:

```css
/* ── Terminal image preview ── */
```

Class names use `terminal-image-` prefix (not `chat-image-`) to avoid collision with chat styles while maintaining naming consistency.
