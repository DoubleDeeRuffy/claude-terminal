<objective>
Fix the visual bug where opening a project with no terminals causes the sessions/resume panel to overflow and overlap the top action bar buttons (resume, +, changes, branch). The root cause is `height: 100%` on `.empty-state` and `.sessions-panel` — percentage heights in flex children bypass the flex layout. Replace with `flex: 1; min-height: 0` pattern.

Purpose: Users can access all action buttons (git pull/push/changes/branch, resume, new terminal) when viewing a project that has no open terminals.
Output: Two CSS files updated with flex-based sizing overrides.
</objective>

**Per D-03 and D-04 — CSS-only fix, no DOM restructuring.**

**In `styles/terminal.css`**, add a new rule block for `#empty-terminals` immediately after the `.empty-state-link:hover` rule (after line 1228, before the file explorer section comment):

```css
#empty-terminals {
  flex: 1;
  min-height: 0;
  overflow: hidden;
  height: auto;
}
```

This overrides the generic `.empty-state` class's `height: 100%` with flex-based sizing. The `min-height: 0` is required because flex items default to `min-height: auto` which prevents shrinking below content size. `height: auto` explicitly overrides the inherited `height: 100%` from `.empty-state`.

**In `styles/projects.css`**, modify the existing `.sessions-panel` rule at line 854. Replace `height: 100%` with `flex: 1` and add `min-height: 0`:

Do NOT modify the generic `.empty-state` class — it is shared across multiple panels. The `#empty-terminals` ID selector scopes the fix.
