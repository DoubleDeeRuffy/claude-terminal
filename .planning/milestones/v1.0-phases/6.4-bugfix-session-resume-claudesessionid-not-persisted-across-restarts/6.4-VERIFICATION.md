## Summary

Phase 6.4 goal is fully achieved across both plans. Plan 01 fixed terminal-mode tab persistence (`claudeSessionId` + `cwd` missing from termData at creation). Plan 02 extended coverage to chat-mode tabs in three ways:

1. `ChatView.js` now propagates every session ID change (including post-/clear) back to termData via `updateTerminal` + `saveTerminalSessionsImmediate`, using a change-guard to avoid redundant saves on every SDK message.
2. `createChatTerminal` now includes `claudeSessionId: resumeSessionId` in the termData literal so resumed chat tabs start with their ID already on termData.
3. `createChatTerminal onSessionStart` now sets `data.claudeSessionId = sid` so fresh (non-resumed) chat sessions get their first session ID persisted to termData.

The full session ID loss cycle is closed for both modes: create -> store -> save -> restore -> create.
