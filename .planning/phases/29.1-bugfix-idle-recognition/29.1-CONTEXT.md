# Phase 29.1: Bugfix Idle Recognition Cross-Tab

## Type
Bugfix (inserted as urgent fix between phases 29 and 30)

## Problem
When a tab that is already idle gets incorrectly set to "working" status because another tab in the same project starts working. It appears that the first tab in the project gets set to working even though Claude didn't do anything in that tab.

## Expected Behavior
Each tab's idle/working status should be independent. Only the tab where Claude is actually producing output should transition to "working". Other tabs in the same project that are already idle should remain idle.

## Likely Root Cause Area
The idle recognition system likely broadcasts status changes at the project level rather than the individual terminal level, causing all terminals in a project to inherit the status of whichever terminal becomes active.

## Files to Investigate
- `src/renderer/state/timeTracking.state.js` — heartbeat/idle logic
- `src/renderer/ui/components/TerminalManager.js` — terminal activity detection
- `src/renderer/events/index.js` — Claude event hooks that trigger status changes
