# Plan 29.1A: Fix cross-tab idle status contamination

## Goal
Fix `wireTerminalStatusConsumer()` so that CLAUDE_WORKING and SESSION_END hook events update only the correct terminal tab — the one actually running Claude — instead of whichever terminal happens to be first in iteration order.

## Root Cause
In `src/renderer/events/index.js`, `wireTerminalStatusConsumer()` (lines 260-280) uses `resolveTerminalId(projectId)` to find the terminal to update. This function simply iterates the terminals Map and returns the **first** terminal matching the project ID — which may be an idle tab that has nothing to do with the active Claude session.

Other consumers in the same file (`wireSessionIdCapture`, `wireTabRenameConsumer`, `wireClaudeActivityConsumer`) correctly use `lastActiveClaudeTab.get(projectId)` to target the right terminal. The status consumer is the only one that doesn't.

## Changes

### File: `src/renderer/events/index.js`

**Change 1: CLAUDE_WORKING handler (line 265)**

Replace:
```javascript
const terminalId = resolveTerminalId(e.projectId);
```

With:
```javascript
const terminalId = lastActiveClaudeTab.get(e.projectId) ?? findClaudeTerminalForProject(e.projectId);
```

**Change 2: SESSION_END handler (line 276)**

Replace:
```javascript
const terminalId = resolveTerminalId(e.projectId);
```

With:
```javascript
const terminalId = lastActiveClaudeTab.get(e.projectId) ?? findClaudeTerminalForProject(e.projectId);
```

## Why This Works
- `lastActiveClaudeTab` is a Map<projectId, terminalId> that gets updated whenever the user clicks/activates a terminal tab (`notifyTabActivated()`, called from `setActiveTerminal()`)
- It tracks which terminal in each project was most recently focused — exactly the one where the user is interacting with Claude
- `findClaudeTerminalForProject()` is the fallback: it returns the highest-ID Claude terminal for the project (newest), which is a better heuristic than "first in iteration order"
- This matches the exact pattern already used by `wireSessionIdCapture` and `wireTabRenameConsumer`

## Scope
- 1 file changed: `src/renderer/events/index.js`
- 2 lines modified (same pattern applied to both handlers)
- No new functions, no new state, no behavioral change for single-terminal projects

## Verification
- `npm test` passes
- `npm run build:renderer` succeeds
- Manual test: open 2 terminals for same project, let one go idle, start Claude in the other → idle tab stays idle
