# Phase 7.1: Fix Hotkeys Settings Toggles - Context

**Gathered:** 2026-02-25
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix the terminal shortcut toggle switches in the Settings/Shortcuts panel so they render as styled toggle switches instead of raw HTML checkboxes. This is a CSS class mismatch bug from Phase 7 — the HTML uses `.terminal-shortcut-toggle` but the toggle CSS targets `.toggle-option`.

</domain>

<decisions>
## Implementation Decisions

### Fix approach
- Reuse the existing `.toggle-option` class from settings.css (lines 62-110) instead of creating new CSS
- Change the HTML label class in ShortcutsManager.js from `terminal-shortcut-toggle` to `toggle-option`
- The `.toggle-option` class handles: hiding the checkbox input, styling the `.toggle-slider` span as a pill toggle, and the checked state transition

### Visual consistency
- Toggles must look identical to other settings toggles — same size (44x24px), colors (accent on checked), and animation
- No special styling for terminal shortcut toggles — they're just settings toggles

### Scope
- Only the terminal shortcuts section is affected — no other toggles are broken
- Only class name change needed — no new CSS required

### Claude's Discretion
- Whether to also rename related JS selectors (`.terminal-shortcut-checkbox`) for consistency, or leave them as-is since they're only used in JS event handlers

</decisions>

<specifics>
## Specific Ideas

- The raw checkbox is visible because `.toggle-option input[type="checkbox"] { display: none }` doesn't match the current `.terminal-shortcut-toggle` parent
- The circle on the right is the `.toggle-slider` span rendering without proper checked-state styling because `.toggle-option input:checked + .toggle-slider` doesn't match

</specifics>
